**1. Анализ A/B-теста финансового приложения для инвесторов**

**Файл**: finance_ab_test_2026_02.ipynb

**Путь**: Practicum_projects/projects/finance_ab_test_2026_02.ipynb

*Описание проекта*:

**Цель проекта**: провести анализ результатов A/B-теста для финансового приложения инвесторов, чтобы оценить влияние обновлённого онбординга на поведение пользователей и их инвестиционную активность.

**Задачи проекта**:

1 Анализ исторических данных клиентов: 
- динамика привлечения, сегментация пользователей и ключевые этапы воронки
- расчет метрик (ключевая метрика - средняя сумма всех депозитов на одного пользователя (включая тех, кто установил приложение или открыл веб-версию.  Барьерная метрика — конверсия из регистрации в первый депозит. Вспомогательная метрика 1 — конверсия из первого депозита во второй. Вспомогательная метрика 2 — средняя сумма всех депозитов на пользователя, который открыл хотя бы один депозит).
  
2 Изучение результатов эксперимента:
- сравнение поведения пользователей в контрольной и тестовой группах, оценка статистической значимости изменений
- исследование влияния нового онбординга на поведение платящих пользователей методом бутстрапа.

**План проекта**:
- Загрузка исторических данных и их предобработка
- Исследовательский анализ исторических данных
- Исследование результатов А/В эксперимента
- Анализ изменений суммы депозитов на платящего пользователя
- Выводы

**Используемые библиотеки** – pandas, numpy, scipy.stats, statsmodels, matplotlib, seaborn.

**2. Расчет Юнит-экономики и метрик мобильного приложения** 
**Файл**: unit_economics_analysis_2026_02.ipynb
**Путь**: Practicum_projects/projects/unit_economics_analysis_2026_02.ipynb

*Описание проекта*:

**Цель проекта**: провести анализ юнит-экономики развлекательного приложения и выявить причины стагнации выручки после запуска нового спортивного контента и изменения маркетинговой стратегии.

**Задачи проекта**:
- Провести анализ юнит-экономики продукта за первые 28 дней после запуска нового контента.
- Выявить причины стагнации выручки после изменения маркетинговой стратегии.
- Определить, какие источники привлечения пользователей приносят прибыль, а какие не достигают уровня окупаемости.
- Разработать и предоставить рекомендации отделу маркетинга для оптимизации стратегии привлечения пользователей.

**План проекта**:
- Загрузка и подготовка данных.
- Когортный анализ пользовательской активности.
- Расчет метрик вовлеченности в продукт
- Расчёт финансовых метрик: CAC, LTV, ROI, Retention за первые 28 дней
- Исследовать метрики в разрезе источника привлечения.
- Рассчитать период окупаемости маркетинговых вложений
- Выводы и рекомендации 

**Инструменты и библиотеки** – pandas, matplotlib, seaborn.
